# local-voice-type

macOS 上で動く、**完全ローカル**の音声入力ツール。
トリガーキーを押している間だけ録音し、離すと自動で文字起こしして、いまカーソルのあるテキストフィールドに挿入します。音声はクラウドに一切送信されません。

クラウド送信のある音声入力サービスの、無料・オフライン・プライバシー安全な代替を目指しています。

---

## 特徴

- **押している間だけ録音**（右Shift）→ 離すと即文字起こし＆挿入
- **完全ローカル**（mlx-whisper / Apple Silicon の GPU・Neural Engine を活用）
- **高速**（large-v3-turbo モデル）
- 日本語の**句読点・漢字変換**に対応、フィラー（「えー」「あの」）も自動で落ちやすい
- **カスタム辞書**で固有名詞・専門用語の精度を補強
- **置換マップ**で人名など「表記を固定したい語」を確定変換
- 録音開始・挿入完了の**効果音フィードバック**
- どのアプリのテキストフィールドにも挿入可能（クリップボード経由）

---

## 動作環境

- **macOS（Apple Silicon / M シリーズ）** … mlx-whisper が Apple Silicon を前提とします
- **Python 3.10 以上**（3.12 で動作確認）

---

## セットアップ

```bash
git clone https://github.com/futosawaguchi/local-voice-type.git
cd local-voice-type

python3 -m venv venv
venv/bin/pip install -r requirements.txt
```

初回起動時に Whisper モデル（large-v3-turbo, 約1〜1.5GB）が自動でダウンロード・キャッシュされます。

## 権限設定（重要）

macOS のセキュリティ上、以下の権限が必要です。
**システム設定 → プライバシーとセキュリティ** で、アプリを動かすホスト（下記参照）に許可を与えてください。

| 権限 | 用途 |
|---|---|
| **入力監視** | トリガーキーの検知（pynput） |
| **アクセシビリティ** | テキスト挿入（Cmd+V の送出） |
| **マイク** | 録音 |

> ホストとは「Python を実行しているアプリ」です。ターミナルから動かすならそのターミナルアプリ、`.app` 化して動かすなら `LocalVoiceType.app` が対象になります。

## 使い方

### そのまま起動して試す

```bash
venv/bin/python local_voice_type.py
```

1. テキストフィールドにカーソルを置く
2. **右Shift を押しながら**話す（録音開始の音が鳴ります）
3. **離す** → 文字起こしされ、挿入されます（完了の音が鳴ります）

## 常駐・自動起動（.app 化）

ターミナルを開かずにログイン時から常駐させるには、軽量な `.app` を作ります。
（torch/mlx は同梱せず、中身は venv の Python を呼ぶだけの薄いラッパーです）

```bash
./build_app.sh          # LocalVoiceType.app を生成
open LocalVoiceType.app # 起動
./stop.sh               # 停止
```

- 初回は **マイク / 入力監視 / アクセシビリティ** の許可を `LocalVoiceType.app` に与えてください。
- **ログイン時の自動起動**：システム設定 → 一般 → **ログイン項目** に `LocalVoiceType.app` を追加。
- `.app` を再ビルドすると署名が変わり、付与済みの権限が無効化されるため**再許可が必要**です。設定の変更は基本 `local_voice_type.py` 側で行えば再ビルド不要です。
- `.app` には実行環境の絶対パスが埋め込まれます。プロジェクトフォルダを移動・削除したら `./build_app.sh` で作り直してください。

## カスタマイズ

### 基本設定

`local_voice_type.py` 冒頭の定数で変更できます。

| 定数 | 既定値 | 説明 |
|---|---|---|
| `TRIGGER_KEY` | `Key.shift_r` | トリガーキー（右Shift）。例: 右Option は `Key.alt_r`、F5 は `Key.f5` |
| `MODEL` | `mlx-community/whisper-large-v3-turbo` | Whisper モデル。精度重視なら `whisper-large-v3` に |
| `LANGUAGE` | `ja` | 認識言語 |
| `MIN_DURATION` | `0.3` | これより短い録音は無視（タップ誤爆対策・秒） |
| `TRAILING_SILENCE` | `0.5` | 末尾に足す無音。文末の句読点を安定させる（秒） |
| `SOUND_FEEDBACK` | `True` | 効果音の ON/OFF |

### カスタム辞書（固有名詞・専門用語）

`dictionary.txt` を作り、1行に1語ずつ書くと、その語に変換されやすくなります（メモリ・速度の犠牲なし）。

```
# 1行に1語。# で始まる行はコメント
Anthropic
リファクタリング
```

### 置換マップ（人名など表記固定）

辞書では安定しない語（珍しい漢字の人名など）は、`replacements.txt` に「誤変換=>正しい表記」を書くと、文字起こし後に確定置換されます。1発話ごとに読み直すので編集は即反映されます。

```
# 誤変換=>正しい表記
わたなべ=>渡邊
```

> `dictionary.txt` と `replacements.txt` は個人の語彙を含むため、リポジトリには含めません（`.gitignore` 済み）。

## トラブルシューティング

- **何を喋っても同じ固定文になる** … 録音が無音です。マイク権限を、いま実行しているホスト（ターミナル or `LocalVoiceType.app`）に付与してください。
- **キーやテキスト挿入が効かない** … 入力監視／アクセシビリティの権限を確認。`.app` を再ビルドした直後は再許可が必要です。
- **メニューバーにアイコンが出ない** … `.app`（薄いラッパー）経由で起動した場合に起きることがあります。機能には影響せず、効果音で代替しています。ターミナルから `python` で直接起動するとアイコンは表示されます。
- **句読点が付きにくい** … `TRAILING_SILENCE` を長めにする／`BASE_PROMPT` を句読点入りの文体に調整。
